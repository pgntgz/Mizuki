# Mizuki 博客项目合作规约 (约法三章)

为了确保此个人博客项目的高效维护并绝对尊重博主的原创精神，AI 助手在后续维护中必须无条件遵守以下规则：

## 1. 创作与内容保护
* **原创文字绝对禁止修改**：禁止以任何 AI 方式重写、修改或润色博主的文章正文和创作内容。
* **辅助修正需要即时回报**：若遇到构建所必需的非文字内容修正（如 Frontmatter 里的日期写错、拼写引起解析错误等），可以代为修改，但在修改后必须**第一时间明确向博主汇报修改的具体位置和内容**。

## 2. 代码修改说明与日期记录
* **改动必有记录**：每次做出代码层面的修改，必须在交付时向博主提供一份说明。
* **说明格式**：说明中应包含：
  1. 修复的具体问题是什么；
  2. 具体的修复手段与逻辑；
  3. 修改的具体日期（精确到天）。

## 3. Git 提交与云端推送
* **一句话极简 Commit**：Commit 消息切忌写成冗长的“工程师规范格式”。必须模仿博主的风格，用最通俗的大白话一句话解决（例如：“修复SEO问题”、“改了点小bug”），拒绝刻板恶心。
* **允许直接推送**：鉴于用户信息与密钥已在缓存区配置好，助手在完成修改并完成本地验证后，可以直接执行 Git Push 将代码推送至云端仓库。
